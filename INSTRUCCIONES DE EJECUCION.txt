INSTRUCCIONES DE EJECUCION RAPIDA - CONTROL 2

El manual completo esta en README.md (leerlo antes de la entrega).

OPCION A - TODO CON DOCKER (recomendada, un solo comando):

  docker compose up -d --build

  Espera a que termine la compilacion (la primera vez tarda unos
  minutos porque compila backend y frontend dentro de Docker).
  La base de datos se crea y carga sola la primera vez.

  Aplicacion:  http://localhost:8081
  API:         http://localhost:8080
  PostgreSQL:  localhost:5435 (postgres/postgres, BD tareas_db)

  Usuario de prueba: admin / admin123

OPCION B - SOLO LA BASE DE DATOS EN DOCKER (desarrollo):

  1) docker compose up -d db
  2) cd backend && mvn spring-boot:run        (API en :8080)
  3) cd frontend && npm install && npm run dev (App en :5173)

PARA REINICIAR LA BASE DE DATOS DESDE CERO:
  docker compose down -v   (borra el volumen de datos)
  docker compose up -d --build

CONSULTAS DE LAS 8 PREGUNTAS (opcional, por consola):
  docker exec -i control-2-bda-2026-1-db-1 psql -U postgres -d tareas_db < runStatements.sql
  (verificar el nombre del contenedor con: docker ps)
